Plans

Our project is based on the creation and utilisation of a ‘smart mirror’. The project’s main function primarily was to provide a more convenient and centralised display of information for users. Typically, users can see this information such as the date, time, reminders, news, and weather reports on their smartphones. However, the smart mirror provides this information without the hassle of going through numerous apps and has the added benefit of reducing screen time in the morning. This in turn allows users to have more time to get ready for the day. Our ‘smart mirror’ project was based on three main features. The first feature was the collection and display of important daily information that included weather reports, current news events, and user-chosen news topics.

This data would be gathered mainly by free APIs that can be easily implemented through the Raspberry Pi. The second feature was that the ‘smart mirror’ would activate upon detection of a user. This could be done through several methods which range from video, sound, light, etc. The main method would most likely be light or motion detection through a video camera device of sorts. As a video camera provides data on user movement and expands the capabilities of the ‘smart mirror’ by making it possible to add more features. Such features as gesture control and providing image data of the user to further assist with advice on skincare and clothing choices.

The third feature was the collection of data to assist the user to determine the best course of action in regard to skincare and clothing choices for any day of the week. The clothing choice feature could provide suggestions on clothing to wear based on the weather data such as temperature, humidity, precipitation and wind speed. Which was previously collected by the weather APIs. For example, Through the weather information, we could give advice such as wearing breathable clothing on hot days or wearing thicker clothes on chilly windy days. The skincare feature would be the biggest side of the project as it is the most overwhelming. As it is very difficult to provide precise and concise information about the overall health of a person’s skin through a simple device such as a ‘smart mirror’. The feature could be done via artificial intelligence that would analyse an image of the person’s skin and based on that AI’s analysis we could output advice and information. However, this would require a lot of research and development, as well as trial and error.

As it is a big undertaking to produce solid, concise and accurate data and advice to match the needs of each user. As we would have to address many different users’ needs and skin types. Apart from the three main functions of our project. Optional auxiliary functions that could be considered AFTER the implementation of the main functions are user customization and personalization. Other optional features for customization or display include widget placement, health information from a compatible device or application, upcoming reminders/appointments, calendars, traffic reports and more. We would undergo this process via the use of more free and open source modules or use a software like Tkinter with python to make a graphical user interface. The user with this would be able to add and customise the information that they wish to see. Furthermore, if we got around to adding LED light to the ‘smart mirror’, we would allow the user the ability to adjust the colour, warmth and light level/brightness of the mirror. This would be done by hooking up the lights to the Raspberry Pi and either programming and linking the lights to a python script or program that allows the user to make all these changes.

If we have enough time two additional features are for the mirror to be able to play sounds from either the monitor or installed external speakers so that it could play music or videos as the mirror could act as a media player of sorts. For the video player, we would need to install a screencast module to play videos and for music, a module such as “MMM-NowPlayingOnSpotify” could be implemented into the Raspberry Pi to play music. However, due to these features, we would now need a connection via WI-FI, Bluetooth or an Ethernet cable running into the Raspberry Pi. Our last feature was the vibrating alarm bed, scrapped due to insufficient funding and time as this was a very ambitious task which would require additional time in coding, R&D and many trials to get working. The whole project is being developed on a Raspberry Pi 3 Model B or A. As for the monitor we were to get a touch screen or just a plain monitor that we could take apart and re-wire to use as the smart mirror. An OLED display would allow us to have a clearer view, as well as make the smart mirror look better. The black on the screen makes the reflection stand out more since blacks on OLED are perfect black instead of a greyish colour that you normally get from a normal LCD panel.

However, this would cost us extra. With the monitor we would also have to of course make the frame and use one-sided material to create the mirror in hand with the monitor.
